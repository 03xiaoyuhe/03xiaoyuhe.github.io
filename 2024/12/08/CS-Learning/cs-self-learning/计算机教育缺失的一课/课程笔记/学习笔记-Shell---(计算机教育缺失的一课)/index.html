关于如何在 window 平台运行 Shell

[!EG] 参考: [[如何在 Windows 10 上下载并安装 Linux Bash Shell？-CSDN 博客]]

一些常用程序的介绍

输入输出类

echo 可以打印出所给参数的内容 (参数就是在命令后用空格分隔的东西) 当我们想传入带空格的参数时，可以用引号将内容扩起来，或者使用 \ 来进行转义 ^d1863a

[!tip] Eg.
"Hello world" or Hello\ world or 'Hello world'

cat 参数所对应的内容添加到输出流中
tail 它可以打印文件后 n 行的信息，使用 -n1 --line1 来表示仅输出最后一行
head 可以打印文件前 n 行信息
tee - read from standard input and write to standard output and files

文件路径操作类

which 使得我们可以获取某个程序的绝对路径
pwd 可以打印出当前工作目录
ls 可以打印当前目录下所有文件的信息，或者指定文件的信息
rm (also remove) 可以删除文件；使用 -r 参数可以进行递归删除，这个指令将会删除所给目录下的所有内容
rmdir 用于删除一个目录，但只能删除空目录 (安全机制)
mkdir 新建一个目录
xdg-open (在 WSL 中需要另外下载) linux 自动匹配合适打开程序从而打开文件
find 用于查找文件，同时也可以在找到后执行一些操作，例如

[!EG] find . -name "*.tmp" -exec rm {} \; 会删掉所有找到的文件

fd 更好的查找程序，可以使用正则表达式，还能忽略搜索 git 文件，可以用不同颜色显示代码，支持 Unicode 编码
locate 为数据创建索引，并定期更新索引。
broot 交互式搜索匹配文件
tree 打印出工作目录的树形结构
nnn 又一个交互式文件夹查看

搜索类

grep 允许我们搜索一段内容
fzf 模糊查找器，交互式的查找内容
zsh 在命令行动态匹配历史命令

字符串操作类

cut 允许我们对输入的字符串进行切割

其他

let 对式子作算数计算。如 let sum=1+1 会计算 1+1 的结果然后赋值给 sum
source 可以从 .sh 文件中加载函数
shellcheck 可以读取脚本并提供警告、语法错误 和 其他可能不正确引用
rg ( ripgrep )
convert 可以处理图像
ffmpeg 可以处理视频
tldr 可以得到一些关于命令的使用案例，更好的使用工具
ag
Corn 用于定期执行任务的工具
history 用于访问历史，history 1 访问指令历史
xargs 命令，它可以使用标准输入中的内容作为参数。 例如 ls | xargs rm 会删除当前目录中的所有文件。

[!tip] 两个有趣的符号 ~ 和 -
~ 会将我们带到主目录 在 windows 中配置 WSL 后主目录即指 user目录
- 会切换当前的目录，通过这个功能我们可以轻易的在两个目录之间来回切换

关于 shell 的输入输出流概念

每一个程序都含有两种流 – 输入流、输出流；默认情况下输入流来源于键盘 (或终端)，在终端输入的任何内容都会进入程序；输出流则是每当程序打印输出时，输出的内容就会传递到这个流中。默认情况下输入输出流都是指向终端的

简单的，我们可以通过尖括号符号 (< >) 来改变输入输出的指向
(如 < file > file : < 表示将程序 (< 前的程序) 的输入重定向为前面那个文件的内容；> 表示将前面程序的输出重定向到后面的文件中

[!tip] Eg.

另外 >> 表示续写而非覆盖 (不同于 > )

[!tip] Eg.

| (管道符 pipe) ：将左边程序的输出作为右边程序的输入

关于权限 $\rightarrow$ 在 ls -l 执行后所列出的关于文件类型及权限的相关解释

[!tip]
d rwx rwx rwx
     $\uparrow$      $\uparrow$      $\uparrow$ 其他所有人
     $\uparrow$      $\uparrow$ 用户组
      $\uparrow$ 文件所有者

第一个字母 d / - 表明该行内容是否为一个目录只有每三个一组的字母：

  • 其中第一个 r / - 表明当前文件对于用户/用户组/其他用户是否有读取权限

  • 第二个 w / - 表明当前文件是否有写入权限

  • 第三个 x / - 表明当前文件是否有执行权限

  • 特别的：对于目录来说 可执行权限 意味着我们是否能够通过 cd 命令访问这个目录

系统小知识

变量创建与取值

创建变量并赋值时：foo=bar 是正确的，而 foo = bar 是错误的，因为在 Bash 中通常用空格作为分隔 程序 变量 等参数的分隔符，同时当我们的文件名中出现空格时，我们通常会用 [[学习笔记-Shell—(计算机教育缺失的一课)#^d1863a|引号 或者 转义 的方式]] (也见 [[第2讲-Shell 工具和脚本 · the missing semester of your cs education#^07ce07|'与"说明]])来避免混淆，取值时使用 ${变量名} 来获得相关值

[!info] 对于在引号内进行赋值的情况，要注意：'' 内的字符串是原义字符，而 "" 内扩起来的就会发生转义

$PATH 可以获取环境变量里配置的内容，细节见 [[第2讲-Shell 工具和脚本 · the missing semester of your cs education#^678023|详细说明]]

  • $0 - 脚本名

  • $1 到 $9 - 脚本的参数。 $1 是第一个参数，依此类推。

  • $@ - 所有参数

  • $# - 参数个数

  • $? - 前一个命令的返回值

  • $$ - 当前脚本的进程识别码

  • !! - 完整的上一条命令，包括参数。常见应用：当你因为权限不足执行命令失败时，可以使用 sudo !!再尝试一次。

  • $_ - 上一条命令的最后一个参数。如果你正在使用的是交互式 shell，你可以通过按下 Esc 之后键入 . 来获取这个值。

  • $( CMD ) - CMD 执行的结果会替换掉 $( CMD )

关于 <( CMD ) （进程替换）

它的作用是：<( CMD ) 会执行 CMD 并将结果输出到一个临时文件中，并将 <( CMD ) 替换成临时文件名。

这个特性，使得我们使用一些 不从标准输入流中 而是 从某些文件中 获得输入的 命令 时更加方便。

同理 当表达式为 {}&&{} 时，如果前一个表达式为 false 时，无论后面表达式为什么值，整个式子都为 false 了，所以也会发生短路这里，我们也可以使用 ; 将两个命令连接起来
