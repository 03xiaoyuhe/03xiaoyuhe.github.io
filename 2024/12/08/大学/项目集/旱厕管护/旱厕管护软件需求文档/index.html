软件需求文档

1. 引言

1.1 目的

本文档旨在定义并详细说明“农村卫生厕所智能管护系统”的软件需求。该系统通过移动互联网和物联网技术，致力于解决农村卫生厕所的管护问题，实现高效、便捷、准确的管理，并推动长效管护机制的建立。

1.2 背景

随着农村卫生改造的深入，需要一套科技先进、智能高效的系统来管理卫生厕所的抽污、维修等工作。本系统旨在整合移动互联网和物联网技术，为村民提供便捷的报抽、报修服务，为管理人员提供有效的任务分派和监控手段。

1.3 文档范围

本文档适用于“农村卫生厕所智能管护系统”的设计、开发和测试过程。涵盖了系统的各个方面，包括移动端、PC端、物联网设备以及相关管理功能。

2. 项目概述

2.1 项目描述

“农村卫生厕所智能管护系统”是一种基于移动互联网和物联网的综合管理系统，旨在解决农村卫生厕所的抽污、维修、改造等问题。系统涵盖了移动端、PC端、物联网设备以及各级管理功能。

2.2 项目背景

近年来，我国农村卫生改造工作逐渐取得显著成效，但在卫生厕所的长效管护方面仍存在一定问题。为提高卫生厕所管理的效率和质量，本系统应运而生。

2.3 业务目标

  • 提供村民便捷的报抽、报修、支付和评价服务。

  • 实现自动化和人工两种模式的任务分派和管理。

  • 建立完善的管护数据统计和分析系统，为决策提供支持。

  • 推动农村卫生厕所的长效管护机制。

3. 术语定义

3.1 专业术语

  • 报抽： 村民申请卫生厕所抽污服务的行为。

  • 报修： 村民申请卫生厕所维修服务的行为。

  • 改厕： 对卫生厕所进行改造升级的过程。

  • 村管员： 负责卫生厕所管理的村级工作人员。

  • 物联网设备： 在卫生厕所中安装的传感器和监测设备，用于实时监控污水液位等信息。

管护端（PC）需求文档

1. 基础数据管理模块

1.1 镇、村管理

  • 实现对镇、村区域基本信息进行管理维护，包括名称、地图坐标等信息。

1.2 维修材料管理

  • 实现对常用维修材料进行统一管理维护，包括材料名称、单价等信息。

1.3 规则配置

  • 实现系统任务预警规则、收费规则、任务分派模式、防误报等规则的管理维护。

2. 户厕信息管理模块

2.1 全局概览

  • 利用图表、文字、数据表格、地图等多种模式，实时展示整个区域内的改厕实际情况，包括改厕户数、改厕覆盖村庄数、各个改厕模式所占比例及数量、历年来改厕数量趋势、改厕详细数据等信息。

2.2 改造基本情况

  • 对户厕改造情况进行统一管理，信息精确到每户，包括门牌号、户主、改造模式、改造时间、改厕照片等信息，可以根据区域、改造模式、时间等信息进行快速查询。

2.3 报抽情况

  • 管理村民报抽详情，根据区域、报抽时间、报抽状态等条件查看报抽详情，内容包括报抽村庄、报抽人、门牌号、联系电话、服务人员、处理结果、完成时间等。

2.4 报修情况

  • 管理村民报修详情，根据区域、报抽时间、报抽状态等条件查看报抽详情，内容包括报修村庄、报修人、门牌号、联系电话、服务人员、处理结果、使用材料、完成时间等。

2.5 村管员管理

  • 对所有村庄的村管员进行统一管理，包括姓名、身份证、手机、管辖村庄等基础信息进行统一添加和维护。还可根据姓名、手机号、负责村庄、身份证等信息进行快速查询检索。

2.6 反馈信息

  • 汇总收集村民日常反馈问题，对反馈问题进行统一管理和维护。反馈内容包括图片、文字信息，对查看人员和阅读状态进行实时反馈。

3. 公厕信息管理模块

3.1 公厕信息

  • 实现对公厕信息统一管理维护，包括公厕所在区域、地址、地图坐标、名称、编号、所属服务站、所有中转站、责任人、联系电话、冲水方式、照片等基本信息。

3.2 管护人员信息

  • 实现对公厕管护人员的统一管理和维护，包括公厕编号、姓名、年龄、性别、联系电话、状态等基本信息。

3.3 公厕液位监测

  • 通过公厕液位监传感器实时监测公厕液位情况，液位达到预警值后系统自动发出预警，并实时发送通知消息通知到相关责任人。还可通过该功能实时查看各公厕液位情况。

4. 管护服务站管理模块

4.1 全局预览

  • 实现整个区域内所有管护站详情进行预览了解，包括各管护站列表及联系人地址、总报抽/报修和今日报抽/报修数量、车辆在线情况、服务满意度情况、工作人员情况、车辆运行轨迹、车辆实时监控、车辆监控

  • 回放等信息，实现所有情况快速预览了解。

4.2 服务站信息

  • 实现对管护服务站统一管理维护，包括所在区域、编号、名称、地址、地图坐标、名称、编号、负责人、联系方式、投入使用日期、配套设施、配套车辆、照片等信息。

4.3 服务站人员信息

  • 实现对服务站人员统一管理维护，包括人员所在服务站编号、工种、姓名、性别、年龄、身份证号、联系电话、状态等基本信息。

4.4 车辆信息

  • 实现对服务站车辆信息统一管理维护，包括所在服务站编号、车牌号、车辆类型、所有人、联系方式、使用性质、品牌型号等基本信息。

4.5 车辆轨迹信息

  • 实现对车辆轨迹统一管理记录，自动记录车辆的形式轨迹，并根据时间、车牌号等进行查询，查询轨迹结果标注在地图上。

4.6 车辆司机信息

  • 实现对所有车辆司机统一管理维护，包括所在服务站、所开车辆车牌号、姓名、性别、年龄、身份证号、是否车辆所有人、入职时间等基本信息。

4.7 新任务提示

  • 当用户报抽或报修后，系统会以语音和系统消息方式，对任务进行自动提醒，如语音播报“您有一个新的报抽（报修）任务，请及时处理”，语音提醒此可定制。

4.8 任务分派

  • 对报抽、报修进行任务分派和任务调度，系统支持自动分派和人工分配两种模式。人工分派需要各级负责人员进行报抽、报修任务分派；自动分派有系统通过分派算法自动进行任务分派。

4.9 任务调度

  • 如区域内报抽、报修任务较多或其他因素，无法在短时间内完成任务的情况下，可以申请任务调度，然后由相关负责人安排其他临近区域内车辆和维修工人完成相关任务。

5. 粪污中转站管理模块

5.1 中转站信息

  • 实现对所有粪污中转站信息统一管理维护，包括所在区域、地址、地图坐标、编号、名称、负责人、联系电话、投入使用日期、占地面积、配套设施、辐射村镇、中转站容积、防渗情况等基本信息。

5.2 中转站人员信息

  • 实现对中转站人员信息统一管理维护，包括所在中转站、工种、姓名、性别、年龄、身份证号、联系电话、入职时间等信息。

5.3 中转站液位监测

  • 通过液位监测仪器实时监测中转站液位情况，达到预警值系统自动发出预警，并通知相关责任人。通过系统可实时查看各中转站液位情况。

6. 巡检抽查记录管理模块

6.1 抽查表管理

  • 实现抽查分类、抽查项目进行标准化管理，进行统一维护管理。

6.2 抽查任务管理

  • 实现自动接收上级派发巡检抽查任务和县（市）区级自发抽查任务的发布、任务分派、抽查结果汇总统一管理和维护。可实时查看巡检抽查进度。

6.3 任务通知

  • 任务发布后，系统自动发送相关消息、巡查计划任务列表到相关干系人。

7. 满意度调查管理模块

7.1 满意度调查

  • 实现对报抽、报修、自定义满意度调查问卷管理维护，发布调查问卷，统计调查问卷结果。

8. 视频监控信息管理模块

8.1 中转站视频监控

  • 对单个或所有中转站实现对中转站实时监控。

9. 智能呼叫中心平台管理模块

9.1 来电弹屏、资料调取

  • 对电话报抽、报修记录进行统一管理维护，用户通过电话报抽、报修时，系统通过呼叫中心自动接入电话信息，可自动关联村户信息。

9.2 来电弹屏

  • 电话打入后，系统自动弹出接入电话界面，自动获取来点号码，根据电话号码自动关联农户相关信息，包括农户基本信息、图片、历史报抽报修记录等。

9.3 智能语音助手

  • 用户打进电话后，自动识别号码，如在改厕档案中有相应号码，可根据语音提示进行自助报抽或报修（举例：按1报抽，按2报修），如无档案自动转接人工或智能留言，留言内容、号码等可通过电脑端或企业微信端查看。

10. 物联网平台一体化管理模块

  • 所有物联网设备均在农厕管护平台中进行集成，统一展示和维护，无须登录其它平台就可使用。

10.1 车载物联网设备

  • 车载设备对车辆实时状态监测数据、车辆运行轨迹状态数据等相关信息进行查询和实时查看。

10.2 中转站视频监控

  • 在平台中实时查看监控设备数据，能够实现视频实时监控、视频回放、视频录制、视频下载。

10.3 中转站液位监测

  • 在平台中能够实时查看中转站化粪池的实时液位数据。

11. 统计报表管理模块

11.1 服务评价统计

  • 实现对所有报抽、报修服务评价统计，根据地区、满意度、时间等信息统计出具体服务评价信息，通过表格和图表进行展示。

11.2 厕所类型统计

  • 实现所有厕所改造模式情况统计，根据地区、改成模式、改厕时间等条件，统计厕所类型数据，通过表格和图表展示。

11.3 材料使用统计

  • 实现所有维修材料使用情况统计，根据地区、材料、时间等条件，统计材料使用情况，通过表格和图表展示。

11.4 车辆作业统计

  • 实现对所有车辆作业情况统计，根据地区、车辆、时间等条件进行数据汇总。

  • 提供表格和图表形式展示，包括车辆作业数量、作业时间、作业区域等信息。

11.5 . 维修作业统计

  • 实现对所有维修人员维修作业情况的统计，根据地区、姓名、时间等条件进行数据汇总。

  • 提供表格和图表形式展示，包括维修人员作业数量、作业时间、作业区域等信息。

11.6. 抽厕作业统计

  • 实现对所有抽厕作业情况的统计，根据地区、户主、报抽时间、完成情况等条件进行数据汇总。

  • 提供表格和图表形式展示，包括抽厕作业数量、作业时间、作业区域等信息。

11.7. 村管员工作排行

  • 实现对村官员工作量的实时统计分析，按工作量进行排名。

  • 提供表格和图表形式展示，包括村管员姓名、工作量、工作时间等信息。

12.1. 大屏数据展示

  • 按各级权限，通过地图+图表形式实时动态展示各区域内的关键数据，包括报抽/报修情况、厕所类型情况、改厕数据情况、车辆在线情况、用户服务评价情况等。

  • 提供当天、本周、本月、本季度、本年等时间维度的大数据分析展示。

12.1. 消息通知

  • 实现定向或全部范围内发送消息通知，包括系统和短信形式。

  • 具备接收最新消息通知的功能，确保管理员及时了解系统的重要信息。

13.1. 权限设置

  • 提供对系统各级使用单位的管理员用户进行统一管理和维护的功能。

  • 包括添加、删除、修改管理员账户的权限。

  • 实现对系统用户角色的统一管理维护，包括添加、删除、修改用户角色。

  • 对角色的系统权限进行统一管理，确保系统安全可控。

管护端（企业微信端）需求文档

1. 报抽管理

1.1 概述

企业微信端的报抽管理模块旨在提供对报抽任务的全面管理，包括任务分配、信息查看和统计分析。

1.2 主要功能

  1. 权限控制： 根据用户权限，显示相应区域内的报抽相关信息和完成情况。

  2. 任务分配： 允许对报抽任务进行分配，确保任务高效完成。

  3. 统计分析： 提供按时间段、区域的报抽任务统计分析功能，帮助用户了解任务完成情况。

2. 报修管理

2.1 概述

企业微信端的报修管理模块用于管理和监控卫生厕所的报修任务，包括任务分配、信息查看和统计分析。

2.2 主要功能

  1. 权限控制： 根据用户权限，显示相应区域内的报修相关信息和完成情况。

  2. 任务分配： 允许对报修任务进行分配，确保任务高效完成。

  3. 统计分析： 提供按时间段、区域的报修任务统计分析功能，帮助用户了解任务完成情况。

3. 任务调度

3.1 概述

任务调度模块允许用户在区域内报抽、报修任务过多，运力和人力不足时，向上级部门发出调度申请，以协调其他区域的车辆和人力。

3.2 主要功能

  1. 调度申请： 允许用户发起调度申请，说明调度原因和需求。

  2. 调度安排： 上级部门收到调度申请后，安排协调其他区域的车辆和人力完成相关任务处理工作。

4. 车辆管理

4.1 概述

车辆管理模块用于监控区域内所有车辆的上线情况和任务执行情况，提供信息查看和统计分析功能。

4.2 主要功能

  1. 车辆上线情况： 查看区域内所有车辆的上线情况，确保足够的运力。

  2. 任务情况查看： 可按区域、时间等条件查看车辆执行的任务情况。

  3. 统计分析： 提供按区域、时间的车辆任务统计分析功能，帮助了解车辆的运行状况。

5. 服务评价

5.1 概述

服务评价模块用于查看区域内所有报抽、报修任务的服务评价情况，提供统计分析功能。

5.2 主要功能

  1. 查看评价信息： 可查看区域内所有报抽、报修任务的服务评价信息。

  2. 按条件统计： 允许按区域、时间、人员、车辆类型等条件进行统计分析，了解服务质量。

6. 消息通知

6.1 概述

消息通知模块用于向特定人员或全部人员发送消息通知，确保信息及时传达。

6.2 主要功能

  1. 消息发送： 允许定向或全部发送消息通知。

  2. 消息接收： 用户可接受最新消息通知，包括系统和短信形式。

7. 问题反馈

7.1 概述

问题反馈模块用于查看和处理问题反馈信息，确保及时解决用户的疑问和问题。

7.2 主要功能

  1. 查看问题反馈： 可查看用户提交的问题反馈信息。

  2. 问题处理： 允许对问题进行处理，跟踪问题解决进度。

工作端（企业微信端）需求文档

1. 消息通知

1.1 概述

消息通知模块用于提醒用户有新消息，确保及时获取相关工作动态。

1.2 主要功能

  1. 消息提醒： 当有新消息时，系统进行提醒，确保用户不会错过重要信息。

2. 待完成任务

2.1 概述

待完成任务模块展示所有待抽任务，提供详细信息和方便的联系方式。

2.2 主要功能

  1. 任务列表： 展示所有待抽任务列表。

  2. 任务详情查看： 可查看任务的详细信息，包括任务内容、时间等。

  3. 一键联系： 提供一键电话联系村民功能，方便沟通。

3. 已完成任务

3.1 概述

已完成任务模块用于展示所有已完成任务，按照完成时间排序，提供查看任务详情和服务评价功能。

3.2 主要功能

  1. 任务列表： 展示所有已完成任务，按完成时间排序。

  2. 任务详情查看： 可查看已完成任务的详细信息。

  3. 服务评价： 提供查看用户对任务的服务评价。

4. 超时提醒

4.1 概述

超时提醒模块用于提醒用户超出规定时间未完成的任务，通过消息和颜色进行警示。

4.2 主要功能

村民端需求文档

1. 依托平台

1.1 概述

村民端通过微信扫描二维码依托平台，实现一系列功能，包括报抽、报修、付费、评价、调查问卷、在线反馈和服务记录。

1.2 主要功能

  1. 二维码扫描： 村民通过微信扫描二维码进入平台，实现便捷的报抽、报修、付费等操作。

2. 报抽

2.1 概述

报抽模块允许村民通过一键式报抽，将报抽信息传递到对应抽粪车工作人员的车辆端和各级管控端。系统具有自动防误报提醒功能，系统管理员可通过后台配置。

2.2 主要功能

  1. 一键式报抽： 提供村民一键式报抽的操作，信息自动传递到相关工作人员和各级管控端。

  2. 自动防误报提醒： 在30天内多次上报时，系统提供自动防误报提醒功能。

3. 报修

3.1 概述

报修模块允许村民通过一键式报修，将报修信息传递到对应维修工作人员和各级管控端。系统具有自动防误报提醒功能，系统管理员可通过后台配置。

3.2 主要功能

  1. 一键式报修： 提供村民一键式报修的操作，信息自动传递到相关维修工作人员和各级管控端。

  2. 自动防误报提醒： 在30天内多次上报时，系统提供自动防误报提醒功能。

4. 付费

4.1 概述

付费模块允许村民调取扫码支付进行付费。

4.2 主要功能

  1. 扫码支付： 提供调取扫码支付的功能，确保村民能够方便快捷地完成付费操作。

5. 评价

5.1 概述

评价模块允许村民对已经完成的服务（抽厕、维修）进行服务评价和建议。

5.2 主要功能

  1. 服务评价： 村民可以对已完成的服务进行评价，评价数据实时显示在管控端。

6. 调查问卷

6.1 概述

调查问卷模块允许村民实时接收和完成上级部门下发的调查问卷，并自动发送到上级管控端进行数据汇总和统计。

6.2 主要功能

  1. 接收和完成问卷： 村民可以实时接收和完成由上级部门下发的调查问卷。

7. 在线反馈

7.1 概述

在线反馈模块允许村民通过扫码进行“建议”、“投诉”、“问题”等类型的反馈，所有反馈可在线记录。

7.2 主要功能

  1. 反馈类型： 提供“建议”、“投诉”、“问题”等类型的反馈。

  2. 在线记录： 所有反馈可在线记录，包括类别、日期、内容和处理状态等。

8. 服务记录

8.1 概述

服务记录模块允许村民的报抽报修信息在线记录，方便查询和追溯。

8.2 主要功能

  1. 在线记录： 报抽报修信息能够在线记录，包括时间、状态、评价及详情。
