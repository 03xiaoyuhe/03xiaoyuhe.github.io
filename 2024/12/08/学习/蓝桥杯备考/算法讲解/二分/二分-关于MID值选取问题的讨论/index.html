因为，c++中的整数除法是向 0 取整的。此时，取中函数会有两种写法 mid = (l+r)/2 以及 mid = (l+r+1)/2

对于 写法一 ，会向前选取，即当区间长度为偶数时，会选择处于中间两元素的前一个，举个例子：l = 1 r = 4 时，此时位于终点的两个下表分别为 $2,3$ ，而 $\lfloor (1+4)/2 \rfloor = 2$ ，由这个例子可以看出 写法一 mid = (l+r)/2 会向前取。对于 写法二 ，毋庸置疑，会向后选取

我们再看一下递归过程，我们发现：

  • 对于问题一来说，处于区间第一个数一定符合题意，而其后则不然；

  • 对于问题二来说，则刚好相反，处于区间终点的数一定符合题解。

为什么要考虑这个情况呢？

因为我们在调试的过程中发现，在递归达到终点，结束之前存在一个状态：即区间长度恰好为二的情况。

此时情况有些特殊，因为我们发现如果此处处理不好，无法愉快的进入区间长度为一的情况时，等待我们的将会是 TLE /dog。

区间长度为 2 时，如何在二者之间选择下一个 Mid 用于 check，进而再次缩小范围进入区间长度为 1 ，然后愉快结束递归就显得尤为重要。

上面我讨论过对于问题一，第一个元素一定符合要求，那我们是不是只需要 check 第二元素就可以缩小范围了。所以我们选择 mid = (l+r+1)/2 。因为，该表达式总是向后选取，若第二个元素符合要求，则 l = mid (l 后移了，因为 mid 选择的是 l 与 r 之间的后者)；若不符合 r = mid-1(r 前移了，因为 mid 原来等于 r )

对于问题二则反之。

总结

通过以上讨论，我们发现二分查找的一个一定要检查的地方——即，区间长度为二时，算法是否还能正常工作到区间长度变一。

故，写二分时一定要检查 区间长度为二的特殊情况
