代码

基础原理解析

依次遍历每一个数，并通过当前数乘以已筛出的素数的方式，向后筛数，保证当前访问到的数一定是被处理过的。

如当我们访问到 6 时，因为 3 一定被访问过，且 $2*3=6$ (前一个 2 为已找出的素数，后一个 3 为当时操作的数字)，所以 6 在 3 被访问时就筛去了。

所以，首先访问到的数就已经被筛过了

进阶原理解析

我们知道，欧拉筛 相较于 埃及筛 来说更快，应为欧拉筛能保证一个数只被筛选一次，而埃及筛则不然，如埃及筛在处理如 $22 = 2*11$ 的时候，就出现在 2 的时候被筛一次，在 11 的时候又被筛一次的情况。

但是欧拉筛是如何保证每一个数只被筛一次的呢，

关键点在 if(i % prime[j] == 0) break;

我们看到基础原理解析的部分里的例子，我们发现欧拉筛其实相当与是在使用这个式子
$$
a = q\cdot b
$$
其中 $a$ 为我们正在筛选的数字，$b$ 为我们在第一层循环访问到的数字，而 $q$ 既是我们当前找到的所有素数之一，同时也是 $a$ 的最小质因数。

那为什么要写 if(i % prime[j] == 0) break; 这一段代码呢

我们看到，如果第一层循环访问到的数字，可以整除当前使用的质因数时，则有
$$
b = q\cdot b’
$$
例如，$4 = 2\cdot 2$ ，当我们访问到 $4$ 时，此时我们的质因数库中已经存入了 $2\ 3$ 两个质数，如果在 $8 = 2 \cdot 4$ 时不跳出循环，接着进行 $12 = 3 \cdot 4$ 此时我们可以马上发现，3 并不是 12 的最小质因数，所以在 $12 = 2 \cdot 6$ 时 12 还会被再筛一次。

所以，在 if(i % prime[j] == 0) break; 后及时止损，跳出循环，防止重复筛数。

前缀和与差分理解

前缀和

相当于一个元素，其前 n 项的和及前缀和

差分

对于一个数组来说，相邻两项的差，构造出的数组即为差分

前缀和 与 差分的关系

当我们知道某数组 (设为 a) 的前缀和数组 (设为 Sum) 时，有 Sum[i] - Sum[i-1] = a[i],及前缀和的差分为 原数组。

同时我们发现，但数组中某一元素增大 c 时，所有包含本项的前缀和都会加 c 。

同时有：
a[i] = Sum[i] - Sum[i-1]
Sum[i] = Sum[i-1] + a[i]

整数拆分问题_C++

一、问题背景

整数拆分，指把一个整数分解成若干个整数的和

如 3=2+1=1+1+1  共2种拆分

我们认为2+1与1+2为同一种拆分

二、定义

在整数n的拆分中，最大的拆分数为m，我们记它的方案数为 f(n,m)

即 n=x1+x2+······+xk-1+xk ，任意 x≤m

在此我们采用递归递推法

三、递推关系

1、n=1或m=1时

拆分方案仅为 n=1 或 n=1+1+1+······

f(n,m)=1

2、n=m时

S1选取m时，f(n,m)=1，即n=m

S2不选取m时，f(n,m)=f(n,m-1)=f(n,n-1)，此时讨论最大拆分数为m-1时的情况

可归纳 f(n,m)=f(n,n-1)+1

3、n<m时

因为不能选取m，所以可将m看作n，进行n=m时的方案，f(n,m)=f(n,n)

4、n>m时

S1选取m时，f(n,m)=f(n-m,m)，被拆分数因选取了m则变为n-m，且n-m中可能还能选取最大为m的数

S2不选取m时，f(n,m)=f(n,m-1)，此时讨论最大拆分数为m-1时的情况

可归纳 f(n,m)=f(n,m-1)+f(n-m,m)

向下取整除的讨论

当存在
$$
\lfloor a/n \rfloor = b \tag{1}
$$ 时是否有
$$
\lfloor a/b \rfloor = n \tag{2}
$$

答案是存在，举个例子，$\lfloor 31/2 \rfloor = 15$ 时，一定有 $\lfloor 31/15 \rfloor = 2$ ，反之不然，如 $\lfloor 31/14 \rfloor = 2$。我们发现 两式的除数分别为 2、15，且前式除数更小。我们得出一个结论，当式 $(1)$ 中 $n < b$ 时，式 $(2)$ 成立。

因为，c++中的整数除法是向 0 取整的。此时，取中函数会有两种写法 mid = (l+r)/2 以及 mid = (l+r+1)/2

对于 写法一 ，会向前选取，即当区间长度为偶数时，会选择处于中间两元素的前一个，举个例子：l = 1 r = 4 时，此时位于终点的两个下表分别为 $2,3$ ，而 $\lfloor (1+4)/2 \rfloor = 2$ ，由这个例子可以看出 写法一 mid = (l+r)/2 会向前取。对于 写法二 ，毋庸置疑，会向后选取

我们再看一下递归过程，我们发现：

  • 对于问题一来说，处于区间第一个数一定符合题意，而其后则不然；

  • 对于问题二来说，则刚好相反，处于区间终点的数一定符合题解。

为什么要考虑这个情况呢？

因为我们在调试的过程中发现，在递归达到终点，结束之前存在一个状态：即区间长度恰好为二的情况。

此时情况有些特殊，因为我们发现如果此处处理不好，无法愉快的进入区间长度为一的情况时，等待我们的将会是 TLE /dog。

区间长度为 2 时，如何在二者之间选择下一个 Mid 用于 check，进而再次缩小范围进入区间长度为 1 ，然后愉快结束递归就显得尤为重要。

上面我讨论过对于问题一，第一个元素一定符合要求，那我们是不是只需要 check 第二元素就可以缩小范围了。所以我们选择 mid = (l+r+1)/2 。因为，该表达式总是向后选取，若第二个元素符合要求，则 l = mid (l 后移了，因为 mid 选择的是 l 与 r 之间的后者)；若不符合 r = mid-1(r 前移了，因为 mid 原来等于 r )

对于问题二则反之。

总结

通过以上讨论，我们发现二分查找的一个一定要检查的地方——即，区间长度为二时，算法是否还能正常工作到区间长度变一。

故，写二分时一定要检查 区间长度为二的特殊情况
